## Project Goal

Bamboo Elf is an AI-native quantitative trading operating environment.

It is not a single trading bot. It is a long-running system that should support:

- market research
- strategy generation
- portfolio allocation
- risk control
- execution
- cycle-based learning
- a native Rust TUI

## Key Locked Decisions

- runtime starts from ZeroClaw copy
- UI uses `ratatui`, not Ghostty
- crypto first
- interfaces should stay ready for future stocks/futures/commodities
- MVP uses local in-process event bus only
- P2P is deferred
- paper trading comes before live trading

## Correct Agent Flow

Use this decision path:

`Research -> Strategy -> Portfolio -> Risk -> Execution`

Important:

- `Portfolio Agent` is mandatory
- `Strategy Agent` must not place orders directly
- risk and execution safety must not depend on LLM availability

## LLM Boundary

High-level rule:

- deterministic code handles safety-critical and execution-critical logic
- LLMs handle synthesis, prioritization, hypothesis generation, and explanation

Practical rule:

- Research and Strategy can use LLMs regularly
- Portfolio and Risk should be mostly deterministic
- Execution should be deterministic on the critical path

## Trading Cycle Model

The system should use cycle-based attention:

- `Scan` - broad universe screening
- `Focus` - deep monitoring of selected assets only
- `Review` - evaluate results and carry learning forward

Open positions must remain monitored even outside the current focus set.
